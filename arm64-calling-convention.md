# ARM64 (AArch64) 函数调用约定

## 1. 寄存器总览

| 寄存器 | 别名 | 用途 | 保存责任 |
|:------:|:----:|------|:--------:|
| x0-x7 | - | 参数/返回值 | 调用者 |
| x8 | - | 间接结果位置 | 调用者 |
| x9-x15 | - | 临时寄存器 | 调用者 |
| x16 | IP0 | 过程调用临时 | 调用者 |
| x17 | IP1 | 过程调用临时 | 调用者 |
| x18 | - | 平台相关 | 平台定义 |
| **x19-x28** | - | **被调用者保存** | **被调用者** |
| **x29** | **FP** | **帧指针** | **被调用者** |
| **x30** | **LR** | **链接寄存器** | **被调用者** |
| SP | - | 栈指针（16字节对齐） | 特殊 |

- 每个 64 位寄存器可按 32 位访问：`xN` → `wN`（低32位，高32位清零）

## 2. 参数传递

### 整型/指针

| 参数位置 | 传递方式 |
|---------|---------|
| 第1~8个 | x0 ~ x7 |
| 第9个起 | 栈（SP+0, SP+8, ...），每个8字节 |

### 浮点

| 参数位置 | 传递方式 |
|---------|---------|
| 第1~8个 | v0 ~ v7（s/d/q 视精度） |

### 返回值

| 类型 | 寄存器 |
|------|--------|
| 整型(≤64bit) | x0 |
| 整型(128bit) | x0:x1 |
| 浮点(32/64/128bit) | s0 / d0 / q0 |
| 大结构体(>16字节) | 写入 x8 指向的地址 |

## 3. FP(x29) 与 LR(x30)

### x29 — 帧指针(FP)
- 指向当前栈帧基址，函数内保持不变
- 构成 FP 链，支持栈回溯和调试
- 局部变量通过 `FP - offset` 访问，栈上参数通过 `FP + offset` 访问

### x30 — 链接寄存器(LR)
- `bl` 指令自动将返回地址写入 x30
- 叶子函数无需保存；非叶子函数必须保存到栈再恢复
- `ret` 等价于 `br x30`

### 典型序言/尾声

```asm
; 序言
stp     x29, x30, [sp, #-16]!    ; 保存 FP/LR，SP -= 16
mov     x29, sp                  ; 建立帧锚点
sub     sp, sp, #N               ; 分配局部空间(N为16的倍数)

; 尾声
mov     sp, x29                  ; 释放局部空间
ldp     x29, x30, [sp], #16      ; 恢复 FP/LR，SP += 16
ret
```

### 栈帧布局

```
高地址 ──────────────────────────────
       │   上级函数参数（如有）    │
       ├───────────────────────────┤ ← 上级 FP
       │   保存的上一级 FP (x29)   │ ← 当前 FP + 0
       │   保存的 LR (x30)        │ ← 当前 FP + 8
       ├───────────────────────────┤
       │   被调用者保存寄存器      │ ← FP + 16 起
       │   局部变量                │ ← FP - N 起
       └───────────────────────────┘ ← 当前 FP = 初始 SP + offset
低地址
```

## 4. FP 与 SP 的协作

| 特性 | SP | FP |
|------|----|-----|
| 含义 | 栈顶指针 | 栈帧基址 |
| 变化频率 | 函数内动态变化 | 函数内固定不变 |
| 主要用途 | 分配/释放栈空间 | 稳定锚点，定位栈帧内容 |

**核心模式**：SP 管理动态空间，FP 提供固定偏移访问。没有 FP 时，同一函数内不同位置的 SP 不同，GDB 无法仅凭当前 SP 确定上一级 SP 的位置。

## 5. 调用者保存 vs 被调用者保存

- **调用者保存** (x0-x18, x30, v0-v7, v16-v31)：函数调用后值可能改变，调用者如需保留需自行保存
- **被调用者保存** (x19-x29, v8-v15 低64位)：被调用函数必须保证返回时恢复原值

## 6. GDB Backtrace 原理

三种方法（优先级从高到低）：

| 方法 | 数据来源 | 可靠性 | 条件 |
|------|----------|--------|------|
| DWARF 回溯 | .debug_frame/.eh_frame | 最高 | 需调试符号 |
| FP 链回溯 | 栈内存中的 FP 链 | 高 | 需帧指针 |
| 启发式扫描 | 扫描栈内存找疑似返回地址 | 低 | 无依赖 |

### FP 链回溯原理
```
当前 FP → [上级 FP] → [上上级 FP] → ... → 0
            [LR]        [LR]
```
循环读取 `*(FP+0)` 得到上一级 FP，`*(FP+8)` 得到返回地址。

### 无 FP 时为何困难
SP 在函数执行过程中动态变化，崩溃时仅凭当前 SP 无法确定"处于函数哪条指令"，从而无法计算上一级 SP。DWARF 通过记录每条指令对应的 CFA（Canonical Frame Address）规则解决此问题。

## 7. -fomit-frame-pointer

### 作用
省略帧指针，将 x29 解放为通用寄存器。

### 对比

```asm
; 保留 FP
stp     x29, x30, [sp, #-16]!    ; 必须保存 x29
mov     x29, sp                  ; 设置 FP
; ... 通过 FP 访问 ...
mov     sp, x29
ldp     x29, x30, [sp], #16

; 省略 FP
sub     sp, sp, #16              ; 不碰 x29
str     x30, [sp]                ; 只保存 LR
; ... 通过 SP 访问，x29 可自由使用 ...
ldr     x30, [sp]
add     sp, sp, #16
```

### 关键结论

- **调用约定不变**：ABI 接口完全相同（参数、返回值、对齐规则均不受影响）
- x29 仍是被调用者保存：如果函数内部修改了 x29，仍需保存/恢复
- 区别在于：不再强制将 x29 当作帧指针，仅在实际使用时才保存
- **收益**：多一个通用寄存器、少几条指令
- **代价**：无 DWARF 信息时调试困难（backtrace 可能丢失中间帧）
- 编译选项 `-O2/-O3` 默认开启，`-O0` 默认关闭

## 8. 与 x86-64 对比

| 特性 | ARM64 | x86-64 System V |
|------|-------|-----------------|
| 整型参数寄存器 | x0~x7 (8个) | rdi,rsi,rdx,rcx,r8,r9 (6个) |
| 浮点参数寄存器 | v0~v7 (8个) | xmm0~xmm7 (8个) |
| 返回值 | x0 / v0 | rax / xmm0 |
| 帧指针 | x29 (FP) | rbp |
| 链接寄存器 | x30 (LR) | 无（压栈） |
| 栈对齐 | 16字节 | 16字节 |
| 红区 | 无 | 128字节 |
